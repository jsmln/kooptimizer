WHY THIS HAPPENED:
══════════════════

1. You created a virtual environment (.venv)
2. Packages were installed into the venv
3. Django runs from the venv (so it works!)
4. But when you open a new terminal, "python" points to global Python
5. Global Python doesn't have the packages
6. You try to install → Goes to wrong location
7. Django still can't find them because it's looking in venv
8. Cycle repeats...
